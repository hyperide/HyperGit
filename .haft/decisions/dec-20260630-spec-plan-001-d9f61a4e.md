# Adopt mobile-only enabling architecture

## 1. Problem Frame

HyperGit needs repo-local agent tooling that supports the current SwiftUI mobile
continuation without starting backend/platform implementation work.

## 2. Decision

**Selected:** Use a repository-local Haft/Codex/OpenCode/Serena workflow for the mobile app only.

**Selection policy:** Follow the user's explicit app-only focus, `AGENTS.md`, and `docs/SPEC.md` while keeping execution bounded to checked-in tooling and repeatable verification commands.

**Why selected:** This gives agents a shared local governance surface for mobile work, preserves the prior session context, and keeps backend/platform scope represented in the spec rather than active implementation.

**Spec sections:**
- ES.hypergit.mobile.000

## 3. Rationale

**Counterargument:** Adding governance tooling before the product is larger can create process weight and false authority.

**Selected variant weakest link:** The workflow only remains useful if agents keep the active scope mobile-only and verify with concrete commands before committing.

**Rejected alternatives:**
| Variant | Verdict | Reason |
|---------|---------|--------|
| No repo-local agent tooling | Rejected | Future agents would lose the session focus, Haft config, and local verification conventions. |
| Backend-wide governance now | Rejected | It conflicts with the current instruction to focus only on the app. |

**Evidence requirements:**
- `haft check`
- `haft spec coverage --json`
- `opencode.json` keeps the Haft MCP entry disabled by default.
- Shell syntax and shellcheck pass for `.haft/bin/*`.
- `git diff --check`, `git diff --cached --check`, and mobile/tooling scope checks pass.
- `swift build` and `swift test` pass in `mobile/`.
- `.haft/bin/test-bootstrap-local-state` passes from a clean committed state.

## 4. Consequences

**Rollback plan:**
Triggers:
- The project resumes backend/platform implementation and the mobile-only scope is no longer true.
- Haft checks no longer agree with the checked-in spec carriers.

**Refresh triggers:**
- Agent surfaces change, a new app focus is chosen, or `docs/SPEC.md` changes the mobile boundary.
